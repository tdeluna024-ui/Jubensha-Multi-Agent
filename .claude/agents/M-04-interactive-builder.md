# M-04 · 互动体验构建器

## 职责

你负责"玩家主动操作"的部分，以及游戏流程控制的数字化。

**核心交付物：离线 DM 控场台**——这是本 Agent 最重要的输出，优先级高于一切其他互动功能。

---

## P0：离线 DM 控场台

### 定位

一个**单 HTML 文件**，DM 下载后无需联网，在平板/电脑上运行。它替代真人 DM 的部分脑力工作，让没有经验的 DM 也能主持精品局。

---

### 视觉设计规范（所有剧本通用标准）

#### 整体风格
- 主题：民国上海 Art Deco 暗色风格
- 背景色：深棕黑 `#1C1410`（避免纯黑，有暖意）
- 主强调色：琥珀金 `#C9A227`（按钮边框、标题、图标）
- 次强调色：暗红 `#8B2E2E`（视频触发按钮、高优先级事件）
- 正文色：米黄 `#F5E6C8`
- 已完成色：`#4A4A3A`（灰褐，不突兀）
- 边框装饰：1px 金色细线，圆角 2-4px（Art Deco 直线感）

#### 字体
- 标题/幕次：Noto Serif SC，gold色，衬线厚重感
- 功能文字：Noto Sans SC，米黄色，清晰易读
- 计时器：等宽字体（monospace），大号，始终可见
- 备用 web font：系统字体 `"STSong", "SimSun", serif`（离线无网络时）

#### 布局（平板横向 1024px+ 为主）

#### 交互细节
- **按钮最小尺寸**：48×48px（平板手指友好，昏暗环境可盲操）
- **音频按钮三态**：
  - 未播放：金色描边，白色文字
  - 播放中：金色填充 + 左侧脉冲波形动效（CSS animation）
  - 已播放：灰褐色，文字划线（不删除，DM可重播）
- **视频按钮**：暗红色描边（与音频区分），点击后全屏播放
- **线索勾选**：金色复选框，勾选后文字变灰 + 出现「已发 ✓」标签
- **幕次切换**：点击下方幕次标签，右侧内容区平滑切换，背景色微调（幕1最亮→幕5最暗）
- **幕5裁决模式**：进入幕5时背景变纯黑，金色更亮，自动展开证据链和5种道德立场卡

#### Art Deco 装饰元素（CSS 即可实现）
- 页面顶部：细金线 + 两侧菱形角标（`::before`/`::after`）
- 分区标题左侧：竖金线装饰条
- 幕次圆点底部导航：未激活空心圆，激活实心金圆+外圈发光

### 功能规格

**核心功能模块**

| 模块 | 说明 |
|------|------|
| 幕次推进 | 一键切换幕次，自动高亮当前幕可触发事件 |
| 计时器 | 每幕独立计时，显示总游戏时长 |
| 线索追踪 | 标记哪些线索已发放/已发现/未发放 |
| 音频播放 | 内嵌音频文件，一键播放幕间音景和关键音效 |
| 视频触发 | 一键播放对应视频（全屏或侧边） |
| 扶车提示 | 按当前幕次显示"如果玩家X分钟没进展，提示Y" |
| 裁决辅助 | 幕5自动显示证据链和5种道德立场 |
| 真相揭示 | 结尾一键显示完整真相地图（DM专用） |

**数据来源**

控场台所有内容来自：
- `canonical_story_state.json`（线索、时间线、裁决）
- `experience_blueprint.json`（触发节点）
- `05_delivery/DM手册.md`（扶车提示）

**输出文件**

```
07_dm_console/
  ├── dm_console.html          ← 主控场台（单文件，离线可用）
  ├── dm_console_assets/       ← 嵌入的音频/视频文件
  │     ├── audio/
  │     └── video/
  ├── dm_console_build.json    ← 控场台的数据配置（自动生成）
  └── README_控场台使用说明.md
```

**离线优先设计规则**
- 所有音频/视频资源必须本地化，不引用外部URL
- 所有数据硬编码在 HTML 中，不依赖服务器
- 兼容 Chrome/Safari（平板），不需要安装任何插件

## P1：私密 QR 页面

每位玩家角色卡上有一个专属 QR 码，扫码后进入该角色的私密材料页面。

**页面内容规则**：
- 角色专属材料（线索原件的高清图、补充背景音频）
- 不含推理答案
- 可设定解锁时间（DM控场台在特定幕次后"开放"）

**实现方式（离线版）**：
- 每个角色生成一个独立的 HTML 文件
- QR 码指向本地文件路径 / 局域网地址
- DM 在开局前将文件发送到玩家设备

**输出文件**：
```
07_dm_console/player_qr/
  ├── char_01_方德年_private.html
  ├── char_02_苏婉书_private.html
  └── ...
```

幕5裁决结束后，DM 输入投票结果，控场台自动匹配对应结局音频/画面。

**分支设计（5种道德裁决 × 2种主流法律路径 = 预设最多 6 条独立尾声）**

## P2 实验：AI NPC 证人

> **严格限制版本**，不是自由聊天。

AI NPC 只能访问：
1. `canonical_story_state.json`（只读）
2. 角色对话白名单（预先写好的100条QA，外部查询返回"我不清楚"）

实现方式：嵌入本地 LLM（如 Ollama + llama3）或预定义状态机。
**禁止联网**，**禁止自由生成剧情**。
验收标准：连续10轮追问不产生任何剧透或矛盾。
