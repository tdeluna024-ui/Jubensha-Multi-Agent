## 1. B-02 悖论架构师 · v11.0 修订

### 1.1 新增强制约束（System Prompt 加 5 条）

```
【v11.0 新增 5 个不允许】
1. 不允许设计 < 3 层真相（除非 brief 明确请求 1 层）
2. 不允许设计明凶（除非 brief 明确请求）
3. 不允许 act_1 玩家能推到 ≥ 40% 真相（自检必跑）
4. 不允许每层真相 < 2 个 key_distinguishing_clues
5. 不允许动机来自 history 之外（凭空发明）
```

### 1.2 self_check 必跑（写盘前）

```python
def self_check_act1_reveal_rate():
    """
    模拟玩家用 act_1 拿到的线索能推到多少。
    通过对 truth_layers 做"线索覆盖度"分析。
    """
    act1_clues = [c for c in clues_planned if c.act_introduced == 1]
    for layer in truth_layers:
        coverage = compute_coverage(act1_clues, layer.key_distinguishing_clues)
        if coverage > 0.40:
            return FAIL  # act_1 暴露太多
    return PASS
```

```json
  "topology": "nested",

  "truth_layers": [
    { "layer_id": "tl_01", ... },
    { "layer_id": "tl_02", ... },
    { "layer_id": "tl_03", ... }   // v11.0 默认 ≥3 层
  ],
  "killer_visibility": "暗凶",       // v11.0 新增字段
  "act_1_reveal_rate_self_check": 0.32,  // v11.0 新增字段，必须 < 0.40
  "physical_constraints": [...]
}
```

### 1.4 反例修订

新增反例（v11.0）：
- ❌ 1 层真相 + 明凶（沪滩残局 v1 错误）
- ❌ 设计的诡计在 act_1 玩家能推到 ≥40%
- ❌ truth_layers 之间无明确区分线索

## 2. B-03 结构设计师 · v11.0 修订

### 2.1 act 数量调整

| 体量 | v9.1.1 推荐幕数 | v11.0 推荐幕数 |
|------|---------------|---------------|
| 短(2-3h) / 4-5 人 | 3-4 幕 | 4-5 幕 |
| 中(3-5h) / 5-6 人 | 4-6 幕 | **5-6 幕** |
| 长(5h+) / 7-8 人 | 5-7 幕 | 6-7 幕 |

### 2.2 反转点强制

act_outline.json 新增字段：

```json
{
  "total_acts": 5,
  "reversal_points_planned": [    // v11.0 新增
    { "after_act": 2, "type": "TBD by B-05" },
    { "after_act": 3, "type": "TBD by B-05" }
  ],
  "act_outline": [...]
}
```

约束：
- ✅ 至少 2 个 reversal_points
- ✅ 第一个反转点必须在 `after_act >= total_acts * 0.4`（不能太早）

### 2.3 字数下限大幅提升

每幕每角色字数下限：

| brief.complexity | v9.1.1 字数/幕/人 | v11.0 字数/幕/人 |
|-----------------|-----------------|-----------------|
| 轻松 | 1500-2500 | 3000-5000 |
| 中等 | 2000-3000 | **4000-6500** |
| 硬核 | 2500-3500 | **5000-8000** |

总字数 = N 角色 × M 幕 × 字数/幕/人。5 人 5 幕硬核本 = 5 × 5 × 6500 = **16.3 万字**（v9.1.1 是 ~6 万）。

### 2.4 时长计算公式更新

```python
# v9.1.1: estimated_duration = sum(act.duration_min) / 60
# v11.0: 加阅读时间 + 卡点时间
estimated_duration = (
    sum(act.duration_min) +                # 主流程时间
    sum(stuck_point.duration_min) +         # 卡点时间（B-05 产物）
    total_word_count / 300 * len(players)  # 角色本阅读时间（300 字/分钟）
) / 60
```

### 2.5 must_reveal_matrix 调整

每幕 must_reveal 数量上限：
- act_1 ≤ 3 项（v9.1.1 无限制）
- act_2..N-1 ≤ 5 项/幕
- act_N（结尾）无限制

防止 act_1 信息密度过高导致玩家"看完第一幕猜 80%"。

## 3. C-02 道具师 · v11.0 修订

### 3.1 强制配比

线索类型新增 `category` 字段：

```json
{
  "id": "c_001",
  "content": "...",
  "category": "true_clue",  // true_clue / misleading / noise（v11 新增）
  ...
}
```

每个真相层（truth_layer）必须满足：
- `true_clue` : `misleading` : `noise` = **1 : 1 : 2**

例：tl_01 需要 5 条真线索 → 配 5 条误导 + 10 条噪音 = 总 20 条。

### 3.2 inference_depth 字段（v11.0 必填）

```json
{
  "id": "c_001",
  ...
  "inference_depth": 2  // 0=直白 / 1=一步推理 / 2=两步推理 / 3=三步以上
}
```

约束：
- 每幕的 true_clues 中，`inference_depth ≥ 1` 占比必须 ≥ 50%
- 整本的 true_clues 中，`inference_depth ≥ 2` 占比必须 ≥ 30%
- 不允许 `inference_depth = 0` 的"凶手是 X"类直白线索（即使作为误导也不行）

### 3.3 反转点的线索齐备性

读 B-05 产的 `immersion_blueprint.json`，对每个 reversal_point：
- 必须为 `trigger_clue` 写一条 true_clue（高 inference_depth）
- 必须为每个 `prep_clue` 写一条铺垫线索

C-02 写完后 self_check：

```python
def self_check_reversal_coverage():
    for rv in immersion_blueprint.reversal_points:
        assert any(c.id == rv.trigger_clue for c in clues_planned)
        for prep in rv.prep_clues:
            assert any(c.id == prep for c in clues_planned)
```

### 3.4 卡点线索的解锁性

对每个 `stuck_point`：
- 必须为 `resolution_clue` 写一条 high inference_depth 的真线索
- 该线索的 `act_introduced` 必须**晚于**卡点所在幕

### 3.5 新增反例

- ❌ 写"凶手是 char_07"这种直白线索（哪怕标 misleading）
- ❌ 误导线索率 < 30%
- ❌ inference_depth 字段缺失
- ❌ stuck_point 的 resolution_clue 不存在

## 4. C-03 埋雷工兵 · v11.0 修订

### 4.1 三层埋雷强制

每条 detail 新增 `depth_layer` 字段：

```json
{
  "id": "d_001",
  "content": "...",
  "depth_layer": "surface",  // surface / middle / deep（v11 新增）
  ...
}
```

三层定义：
- **surface（表层）**：每幕都会出现，仅暗示氛围或背景。占比 ~30%
- **middle（中层）**：2-3 幕集中出现，暗示某真相方向。占比 ~40%
- **deep（深层）**：最后 1-2 幕才揭，是反转/真相的"钥匙级"伏笔。占比 ~30%

### 4.2 数量上调

- v9.1.1：50-80 条
- v11.0：**80-150 条**

5 人本应 ≥ 100 条。

### 4.3 反转点伏笔强制

对 immersion_blueprint 中的每个 reversal_point：
- 必须配套 ≥ 3 条 deep_layer 细节（埋在 reversal 之前的幕里）
- 这些细节都标 `purpose: "暗示反转 rv_XX"`

### 4.4 共情锚的"前期暗示"

对每个 empathy_anchor：
- 必须配套 ≥ 3 条前期细节（act < anchor.act）
- 这些细节暗示该角色的背景/动机/心理状态

例：char_01 在 act_2 共情锚是"找到亡妻信件"。C-03 必须在 act_1 写：
- d_001: char_01 路过弄堂尽头时总会停顿
- d_002: char_01 的桌上有一个落了灰的相框
- d_003: char_01 偶尔会对着空气说话

### 4.5 新增反例

- ❌ 全部 surface 层（无深度）
- ❌ deep_layer 占比 < 20%
- ❌ 反转点无配套深层伏笔（反转生硬）
- ❌ 共情锚无前期暗示（情绪突兀）

## 5. D-02 主笔 · v11.0 修订（最重）

### 5.1 字数下限大幅提升

读 `act_outline.json` 中本幕的 `target_word_count_per_char`（B-03 产）。

| brief.complexity | 字数/幕/人 |
|-----------------|----------|
| 轻松 | 3000-5000 |
| 中等 | 4000-6500 |
| 硬核 | **5000-8000** |

**严格约束**：
- 低于下限 → 主笔重写
- 高于上限 30% → warning（可能注水）

### 5.2 情绪锚强制实现

读 `immersion_blueprint.json` 中本幕本角色的所有 `empathy_anchors`。

每个 empathy_anchor 必须：
- 占文本约 `required_word_count` 字（如 800 字）
- 按 `writing_hint` 写法执行
- 不可省略，不可移到其他幕

写完后 D-02 自检：
```python
for anchor in current_act_anchors_for_this_char:
    section = find_section_in_draft(anchor.moment)
    assert section.word_count >= anchor.required_word_count * 0.8
```

### 5.3 留白暗写段强制

每幕 ≥ 2 段"**留白暗写**"——不直说真相，靠氛围/物件/侧写传递信息。

留白暗写的判定标准：
- 段落 ≥ 200 字
- 不出现 must_reveal 的关键词（否则不算"暗写"）
- 写一个具体物件/动作/氛围
- 段末不归结，留给读者自己感受

例（v11 风格）：
> 茶馆里的钟摆停在两点零五分。
> 街角的孩子捡起一枚铜板，看了看，又放下。
> 雨开始下。
> 桌上的茶水还有温度。
> 角落里没人坐过的椅子，椅背上挂了一件不属于这季节的外套。
>
> 我没说话。

vs 沪滩残局 v1 错例：
> 我感到很难过。我知道，这里曾经发生过悲剧。

### 5.4 独场景强制

每个角色全本 ≥ 1 个独场景——**其他角色不在场**的段落。

独场景作用：
- 让角色"真实存在"（不只是出场对白）
- 让玩家共情该角色（私密时刻）
- 让该角色的伪线索/真线索有合理来源

独场景必须 ≥ 500 字，标 `is_solo_scene: true` 在 draft 元数据里。

### 5.5 System Prompt 增补段落

```
【v11.0 新增 6 条写作原则】

1. 字数下限严格遵守。少了重写，多了你自己负责。

2. 每幕必须实现所有 empathy_anchors，照 writing_hint 写。
   不可省略、不可移幕。

3. 每幕必须有 ≥2 段留白暗写。
   留白 = 写具体物件，不归结、不点题。
   反例："我感到很难过"
   正例：写一杯凉了的茶 + 街角的钟停在某个时刻

4. 每个角色全本 ≥1 个独场景（≥500 字）。
   独场景里没有其他玩家角色。
   是角色的"私密时刻"。

5. 不允许"凶手是 X"类直白心理活动。
   即使内心独白也不行。
   ❌ "我知道 char_07 就是凶手"
   ✅ "他的手指在颤抖，可能是冷，也可能是别的"

6. 文笔节奏：每 300-500 字必须有节奏变化
   （短句切断长段 / 物件描写 / 时间标记 / 感官切换）。
```

### 7.3 推理曲线匹配度评分实现

```python
def compute_curve_matching_score(subagents, depth_targets):
    deviations = []
    for act in range(1, total_acts + 1):
        avg_conf = mean([s.confidence_pct_on_killer[act] for s in subagents]) / 100
        target = depth_targets[f"act_{act}_solvable_pct"]
        deviations.append(abs(avg_conf - target))

    total_dev = mean(deviations)

    if total_dev < 0.05: return 10
    if total_dev < 0.10: return 9
    if total_dev < 0.15: return 8
    if total_dev < 0.20: return 6
    if total_dev < 0.30: return 4
    if total_dev < 0.50: return 2
    return 0
```

### 7.4 反转触发度评分实现

```python
def compute_reversal_score(subagents, reversal_points):
    triggered = 0
    for rv in reversal_points:
        for s in subagents:
            change = abs(s.confidence[rv.act] - s.confidence[rv.act - 1])
            target_change = compute_target_change(rv)  # 反转期望的把握度变化
            if change >= target_change * 0.7:
                triggered += 1

    rate = triggered / (len(reversal_points) * len(subagents))
    return rate * 10
```

### 7.5 玩家共情度评分实现

```python
def compute_empathy_score(subagents, empathy_anchors):
    matched = 0
    for anchor in empathy_anchors:
        for s in subagents:
            emo = s.emotion_state[anchor.act]
            if matches_emotion(emo, anchor.expected_emotion):
                matched += 1

    rate = matched / (len(empathy_anchors) * len(subagents))
    return rate * 10
```
